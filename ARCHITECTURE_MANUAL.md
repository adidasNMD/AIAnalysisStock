## 🧠 二、 双脑架构深度解析

系统中存在两个截然不同的“大脑层（AI 引擎）”，一个负责宏观定性推演，另一个负责微观定量博弈。它们不仅职责独立，还在 `MissionDispatcher` 中相互印证（双大脑共识计算）。

### 1️⃣ 第一大脑：OpenClaw（事件驱动与宏观叙事大脑）
- **语言引擎**：Node.js + TS
- **主要角色**：**系统的总调度者 (Orchestrator)** 和 **战略推演家**。
- **与量化数据的关系**：**完全切断对量化K线数据的直接依赖**。它主要分析的是文本流、新闻、政策、情绪。
- **工作机制**：
  - 采用 **Swarm 多智能体框架**：包含四个核心组件 `DataScout (情报搜集)` -> `LeadAnalyst (事件提纯)` -> `QuantStrategist (产业链推导)` -> `Council (7个角色的市场辩论，含有伪量化声音)` -> `Arbitrator (仲裁总结)`。
  - **信息源头**：高度依赖**文本型数据流**。包括 X/Reddit 的爬虫和 TrendRadar 传入的 HTML 研报总结。
  - **核心输出**：基于“叙事（Narrative）”和“供需链逻辑”推导。比如在遇到“数据中心供电困难”的新闻时，通过分析和推演引申出 `$SMR`（小核电）这种二级关联标的。这是量化模型无法做到的。

### 2️⃣ 第二大脑：TradingAgents（微观量化与决策交易大脑）
- **语言引擎**：Python (运行于端口 8001)
- **主要角色**：**严格遵守风控纪律的虚拟量化交易公司**。
- **与量化数据的关系**：**深度依赖 OpenBB**。所有的分析、看多看空都必须基于真实的市盈率、PEG、技术指标和波动率。
- **工作机制**：
  - 代码基于并扩展于知名项目 `TauricResearch/TradingAgents`。
  - **角色矩阵**：
    - **分析部门**：基本面分析师、技术分析师、情绪分析师、新闻分析师。各个分析师各司其职，去调取相应的数据源。
    - **投研部门**：牛市与熊市研究员基于收集来的数据展开双向辩论。
    - **交易与风控**：Trader 给出进出场条件，Risk Manager 卡死风控红线，最后 Portfolio Manager 做出终极审判（BUY/HOLD/SELL）。
  - **核心输出**：给定一个明确的 Ticker，它负责出具冷血的做多或做空财报评估，防止第一大脑的“理想主义叙事”冲昏头脑买在山顶。

## 🔄 三、 双脑是如何协作的？（Mission Dispatcher 并发逻辑）

位于 `/src/workflows/mission-dispatcher.ts` 的核心中枢机制如下：

1. **探索模式（Explore Mode）- AI 挖掘新机遇的完整链路**：
   - **Step 1 (定性分析与发散)**：输入一条宏观事件，如“某某技术取得重大突破”。第一大脑（OpenClaw）率先进行分析和联想，并推导出一组相关的个股（例如 `$NVDA`, `$AAOI`）。
   - **Step 2 (提取验证目标)**：调度器自动截取第一大脑输出报告中的所有 `Ticker`，将它们剥离出来作为下一步的输入。
   - **Step 3 (定量验证并线跑)**：调度器对这些提取出的 `Ticker`，**同时启动** OpenBB的数据请求与第二大脑（TradingAgents）的定量推演。
   - **Step 4 (共识判决 Consensus)**：调度器汇合两个大脑的立场：
     - 若第一大脑（叙事层面）看涨，且第二大脑（估值层面）也看涨 -> **`agree (强效共识)`，建议立刻建仓**。
     - 若第一大脑看懂叙事，但第二大脑发现该股技术面已严重超买甚至有退市风险 -> **`disagree (严重分歧)`，提示高危风险，观望为主**。

2. **分析模式与复查模式（Analyze & Review Mode）**：
   - 对于 Watchlist 里的存量股票或用户直接指定 `$TSLA` 进行深入分析时，跳过发散环节。双大脑和 OpenBB 会同时开动，从文本热度到基本面给出一个全面的个股“体检报告”。

## ⚙️ 四、 辅助生态基建与网络支持

- **📡 神级情报爬虫：TrendRadar** (Python)
  - 它不受 OpenClaw 控制而**独立自动化运作**的神经元。它每 15-30 分钟默默刷遍全球 40+ 消息接口。使用 `ai_interests.txt` 和 Zhipu 大模型实现信息初筛。它会将真正的“热点”通过独立渠道推给 Dashboard 和飞书群，起到极强的“预警雷达”作用。
- **🗄️ 金融量化总线：OpenBB Platform** (Python)
  - 跑在 8000 端口，是连接外部真实世界（Yahoo Finance 等）的稳定数据网关，为后续的计算提供坚实的数据支持。

> **终极闭环**：TrendRadar 发起信号 ➡️ OpenClaw 定性推理标的 ➡️ TradingAgents 定量校验 ➡️ Dashboard & Telegram 输出最终操作建议。
